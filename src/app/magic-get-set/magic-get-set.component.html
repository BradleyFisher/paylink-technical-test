Magic Get Setter Class.

While this is named MagicGetterSetter to match the test criteria. The properties I have used are based on a simple Person object.

I have used a class for this implementation due to my own understanding of class and interface best practices. I feel that interfaces are best suited for typing an API response for objects which will not be created in the front end.

For example of a CRUD section - a person listItem to display in a table containing a subset of a detail, I would define as an interface. However, a full person object which would be created in the front end to send back to the API - I would create a class due to the power of the constructor.

In this created example, I have set my default properties on creating a new instance of the class as I believe this helps make the class more re-usable.
